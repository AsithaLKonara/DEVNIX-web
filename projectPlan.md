# 🎨 4. UI / UX Direction

You said:

> clean, modern, light theme, background images

Good. Stick to:

### 🎯 Style Guidelines

* Light theme (white / soft gray)
* 1 primary color (blue or violet recommended)
* Lots of spacing (padding is everything)
* Rounded corners (8–16px)
* Soft shadows (not heavy)

---

### 🔤 Typography

* Headings: **Inter / Poppins**
* Body: **Inter / Roboto**

---

### 🧱 Layout Rules

* Max width: 1200–1300px
* Grid-based sections
* Consistent spacing (8px system)

# 🚀 DevNix Web Project – Git Setup Guide

---

# 🧱 1. Initialize the Project

If you already created your project folder:

```bash
git init
```

This starts a new Git repository.

---

# 📂 2. Add Your Project Files

```bash
git add .
```

👉 This stages all files.

---

# 💬 3. First Commit

```bash
git commit -m "Initial commit - DevNix website setup"
```

---

# 🌐 4. Connect to GitHub

Now use your repo:

```bash
git remote add origin https://github.com/AsithaLKonara/DEVNIX-web.git
```

---

# 🌿 5. Set Main Branch

```bash
git branch -M main
```

---

# 🚀 6. Push to GitHub

```bash
git push -u origin main
```

---

# ✅ That’s your exact flow (correct 👍)

---

# ⚠️ Common Mistake (Important)

If your repo already has files on GitHub (like README), you might get an error.

Fix:

```bash
git pull origin main --rebase
git push origin main
```

---

# 📁 7. Recommended .gitignore (VERY IMPORTANT)

Create `.gitignore` file:

```bash
node_modules
.next
.env
.env.local
dist
build
```

---

# 🔁 8. Daily Workflow (Real Usage)

Whenever you make changes:

```bash
git add .
git commit -m "Describe your changes"
git push
```

---

# 🌿 9. Branch Workflow (Pro Level)

Instead of pushing everything to main:

```bash
git checkout -b feature/homepage
```

After work:

```bash
git add .
git commit -m "Homepage layout complete"
git push origin feature/homepage
```

Then create Pull Request on GitHub.

---

# 🧠 10. Clean Commit Messages (Best Practice)

Good examples:

* `Add hero section`
* `Fix mobile navbar`
* `Create services component`

Bad:

* `update`
* `fix stuff`
